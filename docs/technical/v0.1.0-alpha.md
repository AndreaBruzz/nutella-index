# v0.1.0-alpha

## Release Summary

This document captures the first public alpha scope of Nutella Index.

Release tag: `v0.1.0-alpha`

Base package version: `0.1.0`

Status: Alpha (feature-complete MVP, not production-hardened)

## Product Scope in This Alpha

Nutella Index is a localized web app to collect, normalize, and explore Nutella price observations globally.

Implemented user-facing sections:

- Interactive map (`/map`)
- Ranking (`/ranking`)
- Submit data (`/submit`)
- Info hub (`/info`)

Supported locales:

- Italian (`it`)
- English (`en`)

Removed from scope before alpha:

- Blog section

## Feature Inventory

### 1) Interactive Map

- Displays observations from `nutella_index`.
- Uses static ISO-to-coordinate mapping from `lib/coordinates.ts`.
- Shows localized country names where possible.
- Mobile-friendly panel/list interactions.

### 2) Ranking and Full Dataset Visibility

- Country ranking sorted by:
  - Most expensive (`expensive`)
  - Most affordable (`cheap`)
  - Recently updated (`recent`)
- Volatility filter intentionally removed in this alpha.
- Ranking page is section-based: each country expands to show all entries.
- Entry tables expose key fields (without internal IDs):
  - Weight
  - Local price and currency
  - EUR/100g normalized price
  - Collection date
  - Data provider

### 3) Submission Flow

- Public submit form with validation.
- Input normalization for country/currency/email/price/weight.
- API endpoint: `POST /api/submissions`.
- Rate limiting by client IP.
- Submission status starts as `pending` in `user_submissions`.

### 4) Lightweight Analytics

- API endpoint: `POST /api/analytics`.
- Events stored in `analytics_events`.
- Event types in this alpha:
  - `page_view`
  - `form_submit`
  - `language_switch`
  - `form_submit_failure`

## Technical Snapshot

Runtime stack:

- Next.js `16.2.3`
- React `19.2.4`
- TypeScript
- Supabase
- Tailwind CSS v4

Map stack:

- `mapbox-gl`
- `react-map-gl`

Localization:

- App dictionaries under `lib/i18n/`
- Country localization via `i18n-iso-countries`

Data access model:

- Server-side data reads use Supabase admin client for canonical data paths.
- Client-facing writes happen through controlled API routes.

## SEO and Routing Baseline

- Localized sitemap entries for `map`, `ranking`, `submit`, and `info` pages.
- Ranking pages include metadata + OpenGraph/Twitter fields.
- ItemList JSON-LD present on ranking page.

## Known Limitations (Alpha)

- Coordinates are currently maintained in static code, not in DB/admin UI.
- Some data-quality workflows are manual (review/promotion operations).
- No dedicated admin dashboard included in this alpha.
- Performance optimizations are baseline-level (further caching and query shaping expected in later versions).

## Security and Reliability Notes

- Submission endpoint applies rate limiting.
- Submission payloads are validated server-side.
- Analytics insertion failures are non-blocking by design.
- Private submission image handling depends on Supabase storage policies and signed URL workflows.

## What This Alpha Validates

- End-to-end ingestion path from user submission to review-ready storage.
- End-user exploration of normalized global price data.
- Localized UX and locale-aware country naming.
- Dataset transparency through entry-level ranking detail views.

## Out of Scope for v0.1.0-alpha

- Full moderation/admin interface
- Automated geocoding pipeline for all new countries
- Advanced observability dashboards
- Historical trend analytics and forecasting

## Suggested Next Milestone

Target for next release (`v0.2.x`) could include:

- Admin moderation UI
- DB-backed country coordinates
- Operational docs consolidation and runbooks
